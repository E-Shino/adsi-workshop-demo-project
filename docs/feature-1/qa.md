# 打刻メモ機能 — 要件 Q&A

## データモデル

[Question] メモは出勤・退勤それぞれに別々のフィールドを持たせますか？ それとも 1 日 1 レコードに 1 つのメモ欄ですか？
[Answer] 別々のフィールドを持たせる

[Question] メモの最大文字数は何文字にしますか？（例: 100 / 200 / 500）
[Answer] 100文字

[Question] メモは打刻後に編集・削除できるようにしますか？
[Answer] はい。

## UI / UX

[Question] メモの入力タイミングはどうしますか？（A: ボタン押下前にテキスト欄に入力しておく / B: ボタン押下後にダイアログで入力 / C: その他）
[Answer] ボタン押下前にテキスト欄に入力。

[Question] メモが空のまま打刻ボタンを押した場合、確認ダイアログなしでそのまま打刻して良いですか？
[Answer] はい。

[Question] メモ入力欄は単一行（input）で十分ですか？ それとも複数行（textarea）が必要ですか？
[Answer] 単一行でよい。

## 表示

[Question] 打刻履歴一覧（history ページ）でメモをどのように表示しますか？（A: テーブルの列として常に表示 / B: アイコンホバーまたはクリックで展開 / C: 行を展開して詳細表示）
[Answer] C

[Question] ダッシュボードの本日の打刻表示（TodayRecords）にもメモを表示しますか？
[Answer] はい。

## 権限・セキュリティ

[Question] メモの内容は本人以外（上司・管理者）も閲覧できますか？
[Answer] 上司・管理者は閲覧可能

## スコープ確認

[Question] 勤怠修正申請（corrections）にもメモ機能を付けますか？ それとも今回は通常打刻のみが対象ですか？
[Answer] 今回は通常打刻のみ

## 追加確認

[Question] メモを編集できるのは本人のみですか？ それとも上司・管理者も編集可能にしますか？
[Answer] 本人のみ
